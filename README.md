## Концепція проєкту

Adaptive Planner — це вебсистема для планування навчального та позанавчального часу студента. Основна мета проєкту — автоматизувати роботу з розкладом, задачами та календарем, а також допомагати користувачу знаходити найзручніший час для виконання навчальних і особистих справ.

Система поєднує кілька підходів:

- **AI-assisted parsing** — для аналізу складних файлів розкладу.
- **NLP-модель** — для аналізу навчальних задач із файлів або текстових описів.
- **ML-моделі** — для адаптивного планування та оцінки найкращого часу для виконання задач.
- **OR-Tools та алгоритмічна логіка** — для пошуку вільних слотів, перевірки конфліктів і побудови оптимального плану.
- **Google Calendar API** — для синхронізації подій із календарем користувача.

## Ролі AI, NLP, ML та OR-Tools

### AI-assisted parsing для розкладу

AI-assisted parsing використовується для первинного аналізу університетського розкладу, який може бути поданий у різних форматах: PDF, Excel, DOCX, зображення або звичайний текст.

Модуль розпізнавання розкладу має витягувати такі дані:

- назву предмету;
- тип заняття;
- день тижня;
- час початку і завершення;
- групу;
- підгрупу;
- викладача;
- аудиторію.

Користувач вказує свою групу та підгрупу, після чого система залишає тільки ті заняття, які стосуються саме цього студента. Перед імпортом у календар події показуються у preview-таблиці, де їх можна перевірити або відредагувати.

### NLP-модель для навчальних задач

NLP-модель використовується для аналізу текстових описів навчальних задач, методичок, лабораторних робіт або інших файлів із завданнями.

NLP-модуль має визначати:

- назву задачі;
- предмет;
- тип задачі;
- короткий опис;
- ключові слова;
- приблизну складність;
- орієнтовну тривалість виконання;
- можливий дедлайн.

Це дозволяє студенту не створювати всі задачі вручну, а завантажити файл або вставити текст завдання, після чого система автоматично запропонує структуровану задачу.

### ML-моделі для адаптивного планування

ML-моделі використовуються не для самого розпізнавання розкладу, а для адаптивного планування часу.

У системі передбачені такі ML-компоненти:

- **DifficultyModel** — оцінює складність задачі.
- **ProductivityModel** — прогнозує, наскільки конкретний часовий слот підходить користувачу.
- **LoadScoringModel** — оцінює завантаженість дня.
- **SlotScoringModel** — формує підсумкову оцінку вільного слоту.

ML-моделі враховують:

- день тижня;
- час доби;
- кількість пар у день;
- кількість задач;
- складність задач;
- дедлайни;
- історію виконання;
- пропущені задачі;
- вільні часові проміжки.

На основі цих даних система може запропонувати найзручніший час для виконання задачі.

### OR-Tools для оптимізації

OR-Tools використовується для алгоритмічного планування задач і подій.

Його роль:

- знаходити вільні часові слоти;
- не допускати перетину подій;
- враховувати тривалість задач;
- враховувати дедлайни;
- створювати повторювані події;
- генерувати план без конфліктів.

ML-моделі оцінюють якість слотів, а OR-Tools допомагає вибрати оптимальний варіант, який не конфліктує з уже наявним розкладом.

## Кінцевий flow системи

Основний сценарій роботи системи:

1. Користувач авторизується через Google.
2. Завантажує файл університетського розкладу.
3. Вказує свою групу, підгрупу та період семестру.
4. Система аналізує розклад через AI-assisted parsing.
5. Користувач переглядає preview знайдених занять.
6. Після підтвердження події додаються в базу даних і Google Calendar.
7. Користувач створює задачі вручну або імпортує їх із файлів.
8. NLP-модель аналізує текст задач і формує структуровані записи.
9. ML-моделі оцінюють складність задач, завантаження і продуктивність.
10. OR-Tools знаходить можливі часові слоти без конфліктів.
11. Система пропонує найкращі варіанти планування.
12. Користувач підтверджує план.
13. Якщо задача пропущена, система адаптивно переносить її на інший зручний час.
14. Dashboard показує розклад, задачі, дедлайни, навантаження та статистику виконання.
